# 配置

CoCli 使用配置文件来管理仓库和认证信息。配置文件支持 YAML 格式。

## 配置文件位置

CoCli 会按以下优先级查找配置文件：

1. **当前工作目录**：`.qclrc`、`.qcl.yaml`、`.qcl.yml`
2. **用户主目录**：`~/.qclrc`、`~/.qcl.yaml`、`~/.qcl.yml`

## 配置优先级

配置按以下优先级加载（高优先级覆盖低优先级）：

1. **`.qclocal`**（项目目录中的本地配置）
2. **工作区配置**（如果设置了当前工作区）
3. **父级目录的 `.qclrc`**（向上查找）
4. **全局配置**（`~/.qclrc` 或 `~/.qcl.yaml`）

## 配置文件类型

- [`.qclrc`](./qclrc) - 全局或工作区配置文件
- [`.qclocal`](./qclocal) - 项目本地配置文件

## 快速开始

使用 `qcl init` 快速创建配置文件：

```bash
qcl init
```
